9. DOCKER CONFIGURATION

9.1 DOCKERFILE (Multi-stage)
─────────────────────────────────────────
  Stage 1: frontend-build (node:18-alpine) — build React app
  Stage 2: backend-build  (node:18-alpine) — compile TypeScript backend
  Stage 3: production     (node:18-alpine) — runtime
    - Installs sqlite system package
    - Copies compiled backend + built frontend
    - Creates non-root user (nodeuser, UID 1001)
    - Exposes port 3001
    - Health check: curl http://localhost:3001/health (30s interval)
    - CMD: npm start

9.2 DOCKER COMPOSE
─────────────────────────────────────────

    app-dev     — development mode, ports 3001+3000, volume mounts for hot reload
    app-prod    — production mode, port 80→3001, named volume for data
    sqlite-web  — SQLite web UI (coleifer/sqlite-web), port 8080

  Network:       productivity-network (bridge driver)
  Named Volume:  productivity_data (production DB persistence)
