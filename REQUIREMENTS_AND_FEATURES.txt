4. DATABASE SCHEMA (SQLite) — 13 tables
================================================================================

All user-scoped tables include:
  • user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE
  • INDEX on user_id
The categories and tags tables also have UNIQUE(user_id, name).

TABLE: users
  id INTEGER PK AUTO | username TEXT UNIQUE NOT NULL | email TEXT UNIQUE NOT NULL
  password_hash TEXT NOT NULL | api_token TEXT | is_active INTEGER DEFAULT 1
  profile_data TEXT (JSON) | preferences TEXT (JSON)
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP

TABLE: categories                                       [has user_id]
  id INTEGER PK AUTO | user_id INTEGER FK→users(id) CASCADE
  name TEXT NOT NULL | color TEXT DEFAULT '#1976d2' | icon TEXT | description TEXT
  created_at / updated_at DATETIME
  UNIQUE(user_id, name) | INDEX idx_categories_user_id(user_id)

TABLE: tags                                             [has user_id]
  id INTEGER PK AUTO | user_id INTEGER FK→users(id) CASCADE
  name TEXT NOT NULL | color TEXT DEFAULT '#757575'
  created_at DATETIME
  UNIQUE(user_id, name) | INDEX idx_tags_user_id(user_id)

TABLE: todos                                            [has user_id]
  id INTEGER PK AUTO | user_id INTEGER FK→users(id) CASCADE
  title TEXT NOT NULL | description TEXT
  status TEXT DEFAULT 'pending' (pending|in_progress|completed|cancelled|deferred)
  priority TEXT DEFAULT 'medium' (low|medium|high) | due_date TEXT
  category_id INTEGER FK→categories(id) SET NULL
  urgency_level INTEGER DEFAULT 5 (1-10)
  importance_level INTEGER DEFAULT 5 (1-10)
  eisenhower_quadrant TEXT (Q1|Q2|Q3|Q4) | quadrant_auto_assigned INTEGER DEFAULT 1
  bullet_symbol TEXT DEFAULT '•' | time_estimate INTEGER (min)
  energy_required TEXT (low|medium|high) | completed_at DATETIME
  created_at / updated_at DATETIME
  INDEX idx_todos_user_id(user_id)

TABLE: todo_tags (junction)
  todo_id INTEGER FK→todos(id) CASCADE | tag_id INTEGER FK→tags(id) CASCADE
  created_at DATETIME | PRIMARY KEY (todo_id, tag_id)

TABLE: diary_entries                                    [has user_id]
  id INTEGER PK AUTO | user_id INTEGER FK→users(id) CASCADE
  date TEXT NOT NULL (YYYY-MM-DD) | content TEXT
  mood TEXT (great|good|neutral|bad|terrible)
  weather TEXT (sunny|cloudy|rainy|snowy|stormy) | energy_level INTEGER (1-10)
  gratitude TEXT | highlights TEXT | challenges TEXT | tomorrow_focus TEXT
  created_at / updated_at DATETIME
  UNIQUE(user_id, date) | INDEX idx_diary_entries_user_id(user_id)

TABLE: events                                           [has user_id]
  id INTEGER PK AUTO | user_id INTEGER FK→users(id) CASCADE
  title TEXT NOT NULL | description TEXT | event_date TEXT NOT NULL
  event_time TEXT | duration INTEGER (min) | location TEXT
  category_id INTEGER FK→categories(id) SET NULL | bullet_symbol TEXT DEFAULT '○'
  created_at / updated_at DATETIME
  INDEX idx_events_user_id(user_id)

TABLE: bullet_logs                                      [has user_id]
  id INTEGER PK AUTO | user_id INTEGER FK→users(id) CASCADE
  date TEXT NOT NULL | type TEXT NOT NULL (daily|weekly|monthly|yearly|future)
  content TEXT | created_at / updated_at DATETIME
  INDEX idx_bullet_logs_user_id(user_id)

TABLE: blog_categories                                  [has user_id]
  id INTEGER PK AUTO | user_id INTEGER FK→users(id) CASCADE
  name TEXT NOT NULL | slug TEXT NOT NULL | description TEXT | color TEXT | icon TEXT
  parent_id INTEGER FK→blog_categories(id) SET NULL | created_at DATETIME
  UNIQUE(user_id, slug) | INDEX idx_blog_categories_user_id(user_id)

TABLE: blog_posts                                       [has user_id]
  id INTEGER PK AUTO | user_id INTEGER FK→users(id) CASCADE
  title TEXT NOT NULL | slug TEXT NOT NULL | content TEXT
  content_type TEXT DEFAULT 'markdown' | status TEXT DEFAULT 'draft'
  excerpt TEXT | featured_image_path TEXT
  category_id INTEGER FK→blog_categories(id) SET NULL
  reading_time INTEGER | word_count INTEGER | seo_title TEXT
  seo_description TEXT | seo_keywords TEXT | view_count INTEGER DEFAULT 0
  published_at DATETIME | created_at / updated_at DATETIME
  UNIQUE(user_id, slug) | INDEX idx_blog_posts_user_id(user_id)

TABLE: blog_post_tags (junction)
  blog_post_id INTEGER FK→blog_posts(id) CASCADE
  tag_id INTEGER FK→tags(id) CASCADE
  created_at DATETIME | PRIMARY KEY (blog_post_id, tag_id)

TABLE: writing_sessions  [Phase 3 — no UI until Blog]  [has user_id]
  id INTEGER PK AUTO | user_id INTEGER FK→users(id) CASCADE
  blog_post_id INTEGER FK→blog_posts(id) CASCADE
  start_time DATETIME | end_time DATETIME | words_written INTEGER
  productivity_score REAL | notes TEXT | session_type TEXT | created_at DATETIME
  INDEX idx_writing_sessions_user_id(user_id)

TABLE: quadrant_analytics                               [has user_id]
  id INTEGER PK AUTO | user_id INTEGER FK→users(id) CASCADE
  date TEXT NOT NULL | quadrant TEXT NOT NULL (Q1|Q2|Q3|Q4)
  tasks_completed INTEGER DEFAULT 0 | time_spent INTEGER DEFAULT 0 (min)
  productivity_score REAL | created_at DATETIME
  INDEX idx_quadrant_analytics_user_id(user_id)

5. FEATURE SPECIFICATIONS
================================================================================

Features are organized into four phases. Each phase must be fully working
before the next phase begins. Do not stub later-phase features.

  Phase 1 — Core:       Auth, Todos, Categories, Tags, Eisenhower Matrix,
                         Dashboard, Landing, Shared package, Theme, Layout
  Phase 2 — Journaling: Digital Diary, Bullet Journal
  Phase 3 — Content:    Blog system, Blog categories, Writing sessions
  Phase 4 — Insights:   Full Analytics dashboard, Charts, Trends

──────────────────── PHASE 1: CORE ────────────────────

5.1  AUTHENTICATION & USER MANAGEMENT

FEATURES:
  • Registration (username, email, password), login, logout
  • Dev shortcut login (username: dev, password: dev → token: dev-token)
  • API token-based auth (Bearer token); bcryptjs 12 salt rounds
  • Token verification, user profile retrieval
  • Protected routes → redirect to LandingPage when unauthenticated
  • Auto-verify token on app load (localStorage persistence)
  • Login/Signup dialog with tabbed UI; password show/hide toggle
  • Password validation (min 6 chars, confirm match)

UI: LoginDialog, ProtectedRoute

ENDPOINTS:
  POST   /api/auth/register                              (public)
  POST   /api/auth/login                                 (public)
  POST   /api/auth/logout                                [auth]
  GET    /api/auth/verify                                [auth]
  GET    /api/auth/profile                               [auth]

STORAGE: localStorage keys auth_token, user (JSON); 401 auto-clears creds

5.2  TODO MANAGEMENT

FEATURES:
  • CRUD todos with fields: title, description, priority, due date, category,
    urgency (1-10 slider), importance (1-10 slider), time estimate, energy
  • Status: pending, in_progress, completed, cancelled, deferred
  • Auto Eisenhower quadrant from urgency/importance
  • Category assignment, tag support (many-to-many), bullet symbol
  • Search (title/desc), filter (status/priority/quadrant/category), pagination
  • Quick stats chips, completion toggle, status context menu
  • Quadrant preview in add/edit dialog

UI: TodoView (search, filters, stats, list, FAB), TodoList, AddTodoDialog

ENDPOINTS:
  GET    /api/todos                                      [auth]
  GET    /api/todos/:id                                  [auth]
  POST   /api/todos                                      [auth]
  PUT    /api/todos/:id                                  [auth]
  DELETE /api/todos/:id                                  [auth]

BUSINESS LOGIC:
  • urgency ≥ 7 AND importance ≥ 7 → Q1 (Do First)
  • urgency < 7 AND importance ≥ 7 → Q2 (Schedule)
  • urgency ≥ 7 AND importance < 7 → Q3 (Delegate)
  • urgency < 7 AND importance < 7 → Q4 (Eliminate)
  • completed_at set when status → 'completed'

5.3  CATEGORY MANAGEMENT

FEATURES: CRUD categories (name, color, icon, description); auto todo count.
           Used by: todos, events.

ENDPOINTS:
  GET    /api/categories                                 [auth]
  GET    /api/categories/:id                             [auth]
  POST   /api/categories                                 [auth]
  PUT    /api/categories/:id                             [auth]
  DELETE /api/categories/:id                             [auth]

5.4  TAG MANAGEMENT

FEATURES: CRUD tags (name, color); usage count; search by name; get todos for
           tag. Shared across todos and blog posts.

ENDPOINTS:
  GET    /api/tags                                       [auth]
  GET    /api/tags/:id                                   [auth]
  GET    /api/tags/:id/todos                             [auth]
  POST   /api/tags                                       [auth]

5.5  EISENHOWER MATRIX

FEATURES: 4-quadrant visual grid (Q1 red, Q2 blue, Q3 orange, Q4 gray);

UI: EisenhowerMatrixView — 2×2 grid with tasks per quadrant

5.6  MAIN DASHBOARD
